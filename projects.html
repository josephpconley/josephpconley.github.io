Twitter Bots

  • BirdieBot - easily my most popular bot, I had wanted to experiment with my unofficial PGATour datasource. Turns out that the DFS fantasy crowd took a liking to this one. I'm still trying to generalize the concept to give users more power to filter out streams of data (see Atomic Events and PGAStream).

RSS Feeds

I've built quite a few RSS feeds with DataCombinator, and am still doing do on a regular basis. Here are a few of the more useful ones. Most of the ones I've built are based on HTML scraping so there's always a chance that the scraping logic will need to be updated as the target site HTML changes.

  • Github Issues - using a backend Github API integration, I converted the Issues API to an RSS format. Works for any public repo on Github. You have to pass the `org` and `repo` parameters to specify the organization and repository name, the rest of the parameters are the same you'd pass to filter issues on Github. e.g. http://www.datacombinator.com/api/github/issues.rss?org=lagom&repo=lagom&state=open&sort=created
  • Latest NYT Author Articles - I wrote this specifically to follow Andrew Ross Sorkin articles, I like his work and the other RSS feeds purporting to have his latest and greatest weren't working. It's actually a generalized feed for any NYT author, just find their slug id (e.g. Aaron's is `andrew-ross-sorkin`) and plug in to the url, e.g. http://www.datacombinator.com/api/nyt/articles.rss?author=thomas-l-friedman for Thomas Friedman.
